Moongrow has three test layers: TypeScript unit tests for the API, Anchor (Mocha) tests for the on-chain program, and an end-to-end devnet smoke test.

## Anchor program tests

The Anchor test suite uses `ts-mocha` and runs against a local validator.

```bash
# Run all Anchor tests
anchor test

# Run a specific file
npx ts-mocha -p ./tsconfig.anchor.json -t 1000000 \
  programs/moongrow/tests/execute_payout.ts
```

### Test files

| File | Coverage |
|---|---|
| `initialize_store.ts` | Store initialization, duplicate store handling |
| `fund_store.ts` | EURC + SOL funding, insufficient balance |
| `execute_payout.ts` | Fee math, double-pay prevention (12 tests) |
| `withdraw.ts` | Full/partial withdrawal, frozen store rejection |
| `freeze_store.ts` | Freeze + unfreeze, admin-only access |

### What the execute_payout tests verify

- Correct fee split (50 bps = 0.5%)
- Budtender receives `amount - fee`
- Treasury receives `fee`
- `PayoutRecord` PDA is created
- Second call with the same `saleEventId` fails
- Unauthorized caller is rejected
- Frozen store is rejected
- Amount exceeding `max_payout_lamports` is rejected

<Note>
  Anchor tests spin up a local Solana validator automatically. No devnet SOL required.
</Note>

## API unit tests

```bash
# Run API tests
pnpm --filter @moongrow/api test

# Watch mode
pnpm --filter @moongrow/api test:watch
```

Key test areas:
- HMAC signature verification
- Timestamp replay guard
- Campaign commission calculation (FLAT, PERCENTAGE, overlap strategies)
- Sale event deduplication

## Devnet smoke test

The smoke test runs the full payout flow end to end on Solana devnet. The pipeline has been verified end-to-end — POS ingestion through on-chain confirmation.

### Prerequisites

Before running the smoke test for the first time on any environment, the treasury and budtender EURC ATAs must exist on-chain. Run the ATA initializer once:

```bash
HELIUS_RPC_URL=https://devnet.helius-rpc.com/?api-key=YOUR_KEY \
BACKEND_SIGNER_PRIVATE_KEY=YOUR_BASE58_KEY \
MOONGROW_TREASURY_WALLET=YOUR_TREASURY_PUBKEY \
npx tsx packages/api/scripts/init-atas.ts
```

This is safe to re-run — accounts that already exist are skipped. If the ATAs are not initialized, `execute_payout` will fail on-chain with `AccountNotInitialized`.

### Running the test

```bash
HELIUS_RPC_URL=https://devnet.helius-rpc.com/?api-key=YOUR_KEY \
BACKEND_SIGNER_PRIVATE_KEY=YOUR_BASE58_KEY \
MOONGROW_TREASURY_WALLET=YOUR_TREASURY_PUBKEY \
npx tsx packages/api/scripts/test-devnet-payout.ts
```

<Steps>
  <Step title="Creates a fresh EURC test mint">
    Backend signer acts as mint authority — no airdrop needed.
  </Step>
  <Step title="Initializes config PDA">
    Creates or updates the global config with test values.
  </Step>
  <Step title="Initializes a store vault">
    Creates a store with a unique ID seeded by timestamp.
  </Step>
  <Step title="Funds vault">
    Deposits 5,000 test EURC + 1 SOL into the vault.
  </Step>
  <Step title="Calls execute_payout">
    Pays out 1 EURC total and verifies the fee split.
  </Step>
</Steps>
